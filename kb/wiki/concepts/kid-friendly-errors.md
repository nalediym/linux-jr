# Kid-Friendly Errors

> **First seen in:** [commands.md](../sources/linux-jr-story--world--commands.md.md)
> **Also referenced by:** [style.md](../sources/linux-jr-story--world--style.md.md), [CLAUDE.md](../sources/CLAUDE.md.md), [story.yaml](../sources/linux-jr-story--story.yaml.md)
> **Confidence:** high (4 sources agree)

## Definition
Error responses use encouraging, curious language instead of Unix error strings. The command still fails honestly; the *wording* is kid-safe.
— [commands.md](../sources/linux-jr-story--world--commands.md.md)#c-6efe10b6, [CLAUDE.md](../sources/CLAUDE.md.md)#c-ac29fd74

## Details
Example translation: `cat: foo.txt: No such file or directory` becomes *"Hmm, I can't find foo.txt here. Did you `ls` to see what's actually here?"*
— [commands.md](../sources/linux-jr-story--world--commands.md.md)#c-6efe10b6

Unknown commands fail honestly too: *"We don't use that one yet — it's a powerful one. Maybe next season."*
— [commands.md](../sources/linux-jr-story--world--commands.md.md)#c-6efe10b6

The canonical unknown-command response is *"Hmm, I don't know that one yet."*
— [CLAUDE.md](../sources/CLAUDE.md.md)#c-ac29fd74, [style.md](../sources/linux-jr-story--world--style.md.md)#c-c56d105d

The banned-word list (wrong/error/failed/etc.) enforces this at the copy level.
— [style.md](../sources/linux-jr-story--world--style.md.md)#c-ae0d5641, [story.yaml](../sources/linux-jr-story--story.yaml.md)#c-d9dfbe60
