# Style & Voice

> **First seen in:** [style.md](../sources/linux-jr-story--world--style.md.md)
> **Also referenced by:** [story.yaml](../sources/linux-jr-story--story.yaml.md), [CLAUDE.md](../sources/CLAUDE.md.md), [ethos.md](../sources/linux-jr-story--world--ethos.md.md)

## Definition
Copy-level rules that carry the Nick Jr. register and the ethos into every line: short sentences, concrete nouns, active verbs, explicit banned words, explicit always-say phrases, and per-character signature phrases.
— [style.md](../sources/linux-jr-story--world--style.md.md)#c-c0c6d8e6

## Details
**Banned words:** wrong, error, failed, bad, stupid, dangerous, scary, nightmare, terrible, hate, shut up, dumb, loser, weird-as-insult.
— [style.md](../sources/linux-jr-story--world--style.md.md)#c-ae0d5641, [story.yaml](../sources/linux-jr-story--story.yaml.md)#c-d9dfbe60

**Kid-misfire phrases (always say):** "Hmm, I don't know that one yet." / "Let's try another way." / "Not quite — let's have another look." / "Almost!"
— [style.md](../sources/linux-jr-story--world--style.md.md)#c-c56d105d

CLAUDE.md codifies this as an engineering rule: "Never say 'wrong' or 'error' — use encouraging language."
— [CLAUDE.md](../sources/CLAUDE.md.md)#c-96289c50

Ethical-hacker voice rules live inside style.md: briefings start with invitations (never "break into"), after-action acknowledgements ("Nice — you built Sprocket a bed here"), boundary files used rarely, Pip slip-ups modeled warmly.
— [style.md](../sources/linux-jr-story--world--style.md.md)#c-89e93c78, [ethos.md](../sources/linux-jr-story--world--ethos.md.md)#c-d6b03723
